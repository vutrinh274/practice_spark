# spark.practice

A web-based platform for practicing Apache Spark SQL and DataFrame API — built for newsletter subscribers at [vutr.substack.com](https://vutr.substack.com).

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Next.js + Tailwind + shadcn/ui |
| Backend | FastAPI (Python) |
| Execution | Apache Spark Connect 4.0 |
| Database | SQLite (SQLAlchemy) |
| Auth | Clerk (email + password) |
| Payments | Stripe |
| Hosting | Hetzner VPS + Vercel |

## Local Development

### Prerequisites

- Docker (for Spark Connect)
- Python 3.11+ with [uv](https://astral.sh/uv)
- Node.js 20+

### Setup

```bash
# Install all dependencies (Python + Node)
make install
```

### Running locally

Open **3 terminals:**

```bash
# Terminal 1 — Start Spark Connect + seed problems (first time only)
make spark
make seed

# Terminal 2 — Run API (hot reload, port 8000)
make api

# Terminal 3 — Run frontend (hot reload, port 3000)
make web
```

Open `http://localhost:3000`

### Environment variables

Both files are gitignored — contact the repo owner to get the values.

**API** — create `apps/api/.env`:

```bash
cp apps/api/.env.prod.example apps/api/.env
```

| Variable | Purpose |
|---|---|
| `SPARK_CONNECT_URL` | gRPC endpoint for Spark Connect (local: `sc://localhost:15002`) |
| `SPARK_PROBLEMS_DIR` | Path to problems dir as seen by Spark container (`/problems`) |
| `PROBLEMS_DIR` | Path to problems dir on host machine (`../../packages/problems`) |
| `CLERK_SECRET_KEY` | Clerk backend secret — from Clerk dashboard (use Development key locally) |
| `ADMIN_SECRET_KEY` | Secret for admin endpoints (`/admin/subscribers` etc.) |
| `STRIPE_SECRET_KEY` | Stripe secret for subscriber sync (use test key locally) |
| `STRIPE_SYNC_INTERVAL_MINUTES` | How often to sync Stripe subscribers (default: 5) |

**Frontend** — create `apps/web/.env.local`:

| Variable | Purpose |
|---|---|
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk frontend key — from Clerk dashboard (use Development key locally) |
| `NEXT_PUBLIC_API_URL` | Backend API URL (local: `http://localhost:8000`) |
| `NEXT_PUBLIC_CLERK_SIGN_IN_URL` | Sign-in page path (`/sign-in`) |
| `NEXT_PUBLIC_CLERK_SIGN_UP_URL` | Sign-up page path (`/sign-up`) |
| `NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL` | Redirect after sign-in (`/problems`) |
| `NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL` | Redirect after sign-up (`/problems`) |

### Useful commands

```bash
make validate          # validate all problem solutions against the API
make validate PROBLEM=group_by_basics  # validate a single problem
make backup-db         # backup production SQLite database
make logs              # tail production API logs
```

## Authoring Problems

Each problem lives in `packages/problems/<id>/` with this structure:

```
problem.yaml    # config: id, title, difficulty, tags, datasets, validation
problem.md      # description shown to users
fixture.csv     # small dataset (8-15 rows)
reference.py    # reference solution that generates expected.csv
hints/          # concept.md, approach.md, query.md
solutions/      # sql.md, dataframe.md
```

After authoring:
1. Register in `packages/problems/registry.yaml`
2. Add to `make seed` list in `Makefile`
3. Run `make seed` to generate `expected.csv`
4. Run `make validate PROBLEM=<id>` to verify

## Production Deployment

### Architecture

```
User
 ↓ HTTPS
Cloudflare (DNS + SSL termination)
 ↓ HTTP
Hetzner VPS
 └── Docker Compose
     ├── nginx          → routes traffic to API
     ├── api            → FastAPI (pulled from ghcr.io)
     └── spark-connect  → Apache Spark Connect (always running)
         └── SQLite     → named Docker volume (db_data)

Vercel
 └── Next.js frontend → spark.vutrinh.net
```

### CI/CD Pipeline

Every push to `main` that touches backend files triggers **GitHub Actions**:

```
git push main
    ↓
GitHub Actions (.github/workflows/deploy.yml)
    1. Build API Docker image
    2. Push to ghcr.io (GitHub Container Registry)
    3. SSH into Hetzner server:
       - git pull latest code
       - Write .env.prod from GitHub Secret (ENV_PROD)
       - Pull new API image
       - Force-recreate api + nginx containers
       - Run make seed (regenerate expected.csv files)
       - Prune old images
```

Files that trigger backend deploy:
- `apps/api/**`
- `packages/problems/**/fixture*.csv`
- `packages/problems/**/problem.yaml`
- `packages/problems/**/reference.py`
- `packages/problems/registry.yaml`
- `docker-compose.prod.yml`
- `nginx/nginx.conf`

Frontend deploys are handled by **Vercel** automatically when `apps/web/**` changes.

### Required GitHub Secrets

| Secret | Description |
|---|---|
| `SERVER_IP` | Hetzner server IP |
| `SERVER_USER` | `root` |
| `SSH_PRIVATE_KEY` | Private SSH key for GitHub Actions to access server |
| `ENV_PROD` | Full contents of `.env.prod` (all backend env vars) |

### First-Time Server Setup

```bash
# On fresh Hetzner Ubuntu 24.04 server
bash <(curl -fsSL https://raw.githubusercontent.com/vutrinh274/practice_spark/main/scripts/setup_server.sh)
```

See [DEPLOYMENT.md](DEPLOYMENT.md) for detailed step-by-step instructions.

### Rollback

```bash
ssh root@YOUR_SERVER_IP
cd ~/practice_spark
export IMAGE_TAG=PREVIOUS_COMMIT_SHA
envsubst < docker-compose.prod.yml > docker-compose.resolved.yml
docker compose -f docker-compose.resolved.yml up -d --force-recreate api
```

### Database Backup

```bash
make backup-db   # saves to backups/backup_YYYYMMDD_HHMM.sql
```

| User | Access |
|---|---|
| Anonymous | Problems 1-5 (free, submittable) |
